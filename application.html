Model Mobile Application

The first app of the first Decentralized Delivery Network

We are happy to present a rudimentary mobile application that exposes the core functionality of the PAKET protocol. Feel free to use the code available at https://github.com/paket-core/mobile to explore, investigate, and create your own applications.

What can I do with it?

The mobile application makes full use of L1 of the PAKET protocol. This means it allows users to launch packages, promising payment and demanding collateral; to courier packages, depositing collateral and earning payment; and, of course, to receive packages.

The app features a fully functional BUL wallet, supporting the generation and registration of accounts and sending and receiving BUL tokens.

Finally, it also interacts with the PAKET funding server, allowing users to generate and fund Stellar accounts and to purchase BUL tokens using either bitcoin or ether.

What can’t I do with it (yet)?

Currently, the mobile application uses a PAKET operated central routing server as L2 of the protocol. Future versions will support a decentralized routing layer built on a distributed hash table.

The current version does not directly support relaying packages. It is still possible to relay packages by launching a new package which encapsulates an older package. Coming versions will support native relays using a single PAKET smart contract.
